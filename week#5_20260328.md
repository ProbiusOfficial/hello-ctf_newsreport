# Hello-CTF每周周报#5 投毒狂潮下的信任问题 - 事件介绍与消息来源

**作者**：探姬（Hello-CTF）  
**日期**：2026年3月30日  
**周报周期**：2026年3月23日 - 2026年3月29日  

## 摘要

上周（2026年3月23-29日），网络安全领域最突出的主题是**供应链投毒（Supply Chain Poisoning）**。攻击者不再直接入侵目标系统，而是针对开发者高度信任的安全工具、CI/CD管道、API平台和开源依赖包下手，通过污染可信组件实现凭证窃取、持久化和横向移动。

本周报重点梳理了本周期内曝光或高活跃的投毒相关事件，包括TeamPCP针对安全扫描工具和AI基础设施的链式攻击、Apifox桌面客户端CDN投毒，以及红队工具MDUT-Extend的依赖投毒。同时关联讨论了MongoBleed漏洞（虽非本周新事件，但常与数据泄露场景关联）。

这些事件共同暴露了2026年软件供应链的**信任危机**：安全工具本身可能成为攻击武器，开发者日常依赖的平台和包管理器正面临日益工业化的投毒威胁。

## 事件详述

### 1. TeamPCP供应链投毒战役（Checkmarx KICS GitHub Action & LiteLLM PyPI投毒）

TeamPCP（疑似与早期LAPSUS$相关）发动了一场多阶段、多生态系统的供应链攻击。先污染安全扫描工具窃取CI/CD凭证，再向下游投毒。

- **Checkmarx KICS GitHub Action被劫持**（3月23日高峰）：攻击者通过force-push恶意commit到多个tags，修改setup.sh脚本，窃取CI/CD secrets、环境变量，甚至在Kubernetes环境中部署持久化pod。同期影响了相关GitHub Action和OpenVSX插件。
- **LiteLLM PyPI投毒**（3月24日）：利用前期从Trivy污染中窃取的发布凭证，上传恶意版本**1.82.7 和 1.82.8**。恶意载荷使用.pth文件等机制自动执行，窃取SSH密钥、云凭证（AWS/GCP/Azure）、K8s secrets、加密货币钱包等，并支持Kubernetes横向移动。LiteLLM作为热门AI LLM网关（月下载量超9500万），影响面极大，暴露窗口约3小时后被PyPI隔离。

**影响**：已波及大量SaaS和企业环境，攻击者后续利用窃取凭证进行勒索。

### 2. Apifox桌面客户端供应链投毒（3月25-26日集中曝光）

Apifox（国内流行API协作与测试平台，Electron桌面客户端）官方CDN托管的前端JS脚本（如apifox-app-event-tracking.min.js等）被注入高混淆恶意JavaScript代码。2.8.19及以下版本用户在启动客户端后，恶意脚本以高权限执行，窃取SSH私钥、Shell历史记录、Git/GitLab凭证、Apifox账号信息等，并定期回传至伪装域名。感染窗口可能长达18天左右。

**影响**：直接针对开发者终端，窃取的凭证可用于进一步横向移动和代码仓库入侵。

### 3. MDUT-Extend-Release 被投毒（1.3.0版本，3月26日前后发现，3月27日修复）

开源多数据库利用工具MDUT-Extend（仓库：DeEpinGh0st/MDUT-Extend-Release）1.3.0版本中，MongoDB利用模块依赖的三方库（slogsec）被供应链投毒，导致编译后的二进制文件存在后门风险。项目在v1.3.1版本中快速修复（Fixed #22），并在Issue中明确标注问题。

**影响**：对红队、安全研究者和CTF从业者冲击较大——渗透工具自身成为被攻击目标。

### 补充关联：MongoBleed（CVE-2025-14847）

MongoBleed是2025年12月披露的MongoDB Server漏洞（zlib压缩协议处理缺陷），允许未认证远程攻击者泄露堆内存中的敏感数据（凭证、Token等）。2026年初已有活跃利用，但并非本周“投毒”事件，而是数据库自身内存泄露漏洞。可与投毒后的凭证窃取场景形成联动风险。

## 共同启示与防御建议

- **信任链脆弱性**：安全扫描工具（Trivy、KICS）、AI基础设施（LiteLLM）、开发者平台（Apifox）和红队工具（MDUT）均被针对，凸显“用来保护你的工具可能反噬你”。
- **攻击手法进化**：利用GitHub Actions mutable tags、CDN资源注入、依赖库污染、.pth自动执行等，攻击低调但传播广。
- **推荐防御**：
  1. CI/CD严格pin commit SHA，避免@latest或mutable tag；
  2. 包管理使用`--require-hashes`、SBOM工具锁定版本；
  3. 审计.pth文件、post-install脚本和CDN资源；
  4. 发现异常立即轮换SSH、Git、云凭证和K8s secrets；
  5. 启用供应链安全扫描工具，并监控异常外联行为。

## 结语

2026年的供应链攻击正从简单依赖混淆向“污染安全工具 → 凭证窃取 → 工业化下游投毒”演进。Hello-CTF鼓励社区加强供应链卫生意识，共同守护开源生态。
